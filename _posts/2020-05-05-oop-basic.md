컴퓨터 프로그램들을 객체들의 모임으로 파악하고자 하는 패러다임, 객체들은 서로 메시지를 주고 받을 수 있고 데이터를 처리할 수 있다.

객체 지향 프로그래밍의 장점

- 프로그램을 유연하고 변경이 용이하게 한다.
- 프로그램의 개발과 보수를 간편하게 한다.
- 직관적인  코드 분석을 가능하게 한다.
- 강한 응집력과 약한 결합력을 지니게 한다
    - 응집력 (코히전) : 특정 목적을 위해 밀접하게 연관된 기능이 모여서 구현되어 있고 지나치게 많은 일을 하지 않으면 높다고 한다.
    - 결합력 (커플링) : 프로그램 코드의 한 요소가 다른것과 얼마나 연결되어 있고 의존적인지 나타내는 정도.

클래스 : 같은 종류의 집단에 속하는 속성과 행위를 정의한 것

객체 : 클래스의 인스턴스, 상위 클래스의 속성을 지니고 있으며 개별적인 특성과 행위(메소드)또한 갖고 있다. 

메서드 : 클래스로부터 생성된 객체를 사용하는 방법. 객체의 속성을 조작하는데 사용한다.

- 캡슐화 : 모듈화, 정보은닉 → 은닉 : 객체의 데이터 외부에서 접근하는 것을 막고 함수를 통해서만 조작할 수 있도록 한다 / 모듈화 : 독립적으로 역할을 제대로 수행하기 위해 필요한 데이터와 기능을 모은것
- 상속 : 재사용 → 상위 개념의 특징을 하위 개념이 물려 받는것
- 추상화 : 모델링 → 객체들이 가진 공통의 특성을 파악하고 불필요한 특성을 제거한다
- 다형성 : 사용편의 → 모듈이 갖는 정체성과 표현방식, 추상 클래스나 인터페이스를 의미
