#### DL (Dependency Lookup : 의존성 검색)

- 저장소에 저장되어 있는 빈에 접근하기 위하여 개발자들이 컨테이너에서 제공하는 API를 이용하여 사용하고자 하는 빈을 Lookup한다.

#### DI (Dependency Injection : 의존성 주입)

- 각 계층 사이, 각 클래스 사이에 필요로 하는 의존 관계를 컨테이너가 자동으로 연결해주는 것
- 각 클래스 사이의 의존관계를 빈 설정 정보를 바탕으로 컨테이너가 자동으로 연결해준다.
- DL 사용 시 컨테이너 종속성이 증가하여 이를 줄이기 위해 DI를 사용한다.
- 의존하는 클래스에 대한 인스턴스를 직접 생성하지 않고 컨테이너로부터 생성된 빈을 setter 또는 생성자를 통해 외부로부터 주입받는다.
  - 클래스 사이의 의존 관계를 빈 설정 정보를 컨테이너가 자동적으로 연결해준다.
  - 즉 코드 재사용성과 결합도 감소를 위해서 프레임 워크가 객체를 만들고 이에 대한 레퍼런스를 넘겨받는 것을 의미한다.
  - 객체의 생성과 데이터 주입만 담당하는 팩토리에 해당하는 별도의 공간인 IoC 컨테이너에서 객체를 생성하고 데이터간 의존성을 주입한다.
- DI 구현 방법
- 생성자 주입(Constructor Incjection)
  - 생성자를 이용하여 클래스 사이의 의존 관계를 연결
  - 생성자에 파라미터를 지정함으로 생성하고자하는 객체가 필요로 하는 것을 명확하게 알 수 있다.
  - Setter 메소드를 제공하지 않아서 간단하게 필드를 불변 값으로 지정 가능하다
    - 생성자의 파라미터가 많을 경우 코드가 복잡해 보일 수 있다.
    - 조립기 입장에서는 생성의 순설르 지켜야하기에 상당히 불편하다.
    - 스프링 프레임 워크의 빈 설정 파일에서 constructor-arg를 사용한다.
- Setter 주입 (Setter Injection)  : 스프링 추천
  - 인자가 없는 생성자나 인자가 없는 static factory 메소드가 빈을 인스턴스화 하기 위하여 호출된 후 빈의 setter메소드를 호출하여 실체화하는 방법
    - 객체를 생성 후 의존성 주입하는 방식이기에 구현시에 좀 더 유연하게 사용할 수 있다.
    - 세터를 통하여 필요한 값이 할당되기 전까지 객체를 사용할 수 없다.
    - 스프링 프레임 워크의 빈 설정 파일에서 property 사용
- 메소드 주입 (Method Injection)
  - 싱글턴 인스턴스와 논싱글턴 인스턴스의 의존관계를 연결시킬 필요가 있을 경우 사용하지만 많이 사용되지 않는다.
