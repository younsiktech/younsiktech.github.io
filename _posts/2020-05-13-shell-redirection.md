리눅스 쉘은 다음과 같은 표준 입출력을 항상 파일 형태로 열고 있다.

- 표준입력(stdin: 키보드)
- 표준출력(stdout: 모니터)
- 표준에러(stderr: 모니터)

#### I/O 재지향 (redirection)

프로그램은 결과를 출력 장치(모니터, 파일 등)로 넘겨주는데 데이터를 임의로 다른 장치로 보낼 수 있다.

즉, 파일이나 프로그램, 명령어 등의 출력을 다른 파일이나 프로그램, 명령어의 표준 입력으로 보내는 것을 의미한다.

표준 입출력은 파일 형태로 열려 있는데 리눅스에서 열려 있는 파일은 파일 디스크립터(file descriptor)를 할당 받는다.

이렇게 할당받은 디스크립터를 이용해서 재지향을 할 수 있다.

#### 디스크립터 (descriptor)

| 파일             | 파일 디스크립터 |
|:----------------|:------------|
| 표준 입력 (stdin) | 0 |
| 표준 출력 (stdout) | 1 |
| 표준 에러 (stderr) | 2 |

#### 사용법

| 사용법      | 의미 |
|:--------- |:------------|
| \> {파일 이름} | 표준 출력을 파일로 재지향 합니다. 파일이 없으면 새로 만들고, 파일이 있으면 덮어씁니다. |
| \>> {파일 이름} | 표준 출력을 파일로 재지향 합니다. 파일이 없으면 새로 만들고, 파일이 있으면 파일의 끝에 덧붙입니다. |
| 2>&1 | 표준 에러를 표준 출력으로 재지향합니다. 표준 에러도 표준 출력의 자격으로 보내집니다. |
| \< {파일 이름} | 파일로부터 표준 입력을 받도록 재지향합니다. |

#### 예제

- test.txt 파일의 출력을 test.out에 저장
  - cat test.txt > test.out
- aaa.log 파일의 출력을 bbb.log 파일의 끝에 덧붙인다.
  - cat aaa.log >> bbb.log 
- cat 명령어의 에러를 표준 출력으로 재지향해서 out 파일로 재지향한다. (에러 발생시 화면이 아닌 out 파일에 기록된다.)
  - cat sample > out 2>&1
- ls 명령어의 에러를 /dev/null로 재지향한다. (/dev/null은 특수 파일이라 이 파일로 출력되는 데이터는 버려진다.)
  - ls 2> /dev/null
