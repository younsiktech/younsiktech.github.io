- IoC (Inversion of Control)
  - 의존 관계 주입(Dependency Injection)이라고 하며
  - 어느 객체가 사용할 다른 객체를 직접 만들어서(의존) 사용하는 것이 아니라 주입 받아 사용하는 것을 의미함.
  - 의존 관계가 존재하면 테스트가 힘들어짐, 의존성 주입을 통하면 가짜 객체를 만들어서 테스트가 용이해짐.

- 스프링 IoC 컨테이너
  - 역할
    - 빈 인스턴스 생성
    - 의존 관계 설정
    - 빈 제공
    - 애플리케이션 콤포넌트의 중앙 저장소
      - 빈 설정을 읽어 빈을 생성하여 보관한다.
  - 인터페이스
    - BeanFactory 인터페이스 : 스프링 IoC 컨테이너의 최상위 인터페이스
      - 빈의 다양한 라이프 사이클들을 제공
    - ApplicationContext 인터페이스 : 실제로 사용하는 컨테이너
      - BeanFactory 인터페이스를 상속받음, 다른 기능들도 추가됨.
      - ApplicationEventPublisher : 이벤트 발행
      - MessageSource : Internalization(다국어 지원 : i18n)
      - ResourceLoader : 리소스 읽어 들이는 기능

- 스프링 빈 (Spring Bean)
  - 스프링 IoC 컨테이너가 관리하는 객체
  - 과거 xml로 빈을 설정하였으나 현재는 애너테이션을 통한 DI로 POJO 객체를 빈으로 등록할 수 있다.
  - 클래스에 어노테이션(@Service, @Repositry 등)을 붙이면 AutoScan을 통해 빈으로 등록된다.
  - DI를 하려면 빈으로 등록되어 있어야 한다.
  - 스프링에서 오직 하나만 있어야하는 인스턴스는 빈으로 적용한다. (싱글톤으로 사용하고 싶을 때 : 빈 기본 스코프)

- 스프링 빈 스코프
  - 싱글톤 : 하나만 만들어서 사용
    - 메모리적으로 효율적, 컨테이너에 이미 만들어서 있어 런타임시 성능 최적화에 유리
  - 프로토타입 : 매번 다른 객체를 만들어서 사용

- 스프링 빈 라이프사이클
  - 스프링 IoC 컨테이너에 등록된 빈에만 해당
  - 빈의 생성 전후 등에 따라 필요한 작업이 있을 때 사용할 수 있다.
